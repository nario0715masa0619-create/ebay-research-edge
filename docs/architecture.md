# eBay-Research-Edge Architecture

## 1. 全体構成
本システムは以下の4層構造とする。

1. 取得層
2. 整形層
3. 分析層
4. 表示層

## 2. 各層の責務

### 2.1 取得層
役割:
- eBay Sold データ取得
- 国内価格データ取得

責務:
- 検索実行
- 生データ保存
- fetched_at 記録
- 元レスポンス保持

### 2.2 整形層
役割:
- タイトル正規化
- 価格・通貨・送料の整形
- キャラ名 / 商品種別の抽出

責務:
- 表記ゆれ吸収
- 不要語除去
- 欠損値処理
- total_price 統一

### 2.3 分析層
役割:
- 指標計算
- 粗利計算
- スコアリング
- 判定

責務:
- sold_30d / sold_90d 集計
- median / avg / min / max
- STR計算
- estimated_profit_rate計算
- candidate_score計算
- decision_status判定

### 2.4 表示層
役割:
- ユーザー入力
- 指標表示
- 候補一覧表示
- CSV出力

## 3. 処理フロー
1. ユーザーが検索条件を入力
2. eBay Sold データ取得
3. 国内価格データ取得
4. 生データ保存
5. データ正規化
6. 指標計算
7. 候補スコア計算
8. テーブル表示 / CSV出力

## 4. 設計原則
- 取得処理と分析処理を分離する
- 生データは必ず保存する
- 正規化後データは再利用可能にする
- スコア式は後から調整可能にする
- MVPではシンプルな構成を優先する
