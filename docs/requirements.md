# eBay-Research-Edge Requirements

## 1. 目的
日本国内で調達可能な商品について、eBay Sold データと国内価格を比較し、
LIST候補を数値ベースで抽出する日本人セラー向けリサーチ支援ツールを実装する。

## 2. 背景
現状の手作業リサーチには以下の課題がある。

- eBay Sold の確認に時間がかかる
- 国内価格との比較を毎回手作業で行っている
- 判断が感覚に寄りやすい
- LIST候補の優先順位付けが属人的

## 3. 対象ユーザー
- 日本国内から eBay に出品するセラー
- 国内仕入れ → eBay販売 の価格差を狙う個人・小規模事業者
- 完全自動化ではなく半自動の判断支援を求めるユーザー

## 4. MVPで実装する機能
- eBay Sold データ取得
- 国内価格データ取得（まずは1サイトのみ）
- タイトル正規化
- 価格 / 送料 / 通貨の正規化
- 30日 / 90日 Sold 件数集計
- eBay価格の中央値 / 平均値 / 最安値 / 最高値算出
- 国内価格との価格差算出
- 概算粗利率算出
- LIST候補スコア算出
- CSV出力
- 簡易UIまたは簡易ダッシュボード

## 5. MVPで実装しないもの
- 完全自動出品
- 複数国内サイト同時対応
- 高度な画像認識
- 自動仕入れ
- 本格的な競合セラー追跡
- 本番向け大規模インフラ

## 6. 主要ユースケース
### UC-01
ユーザーがキーワードを入力し、eBay Sold の30日 / 90日データを取得する。

### UC-02
国内価格データを取得し、eBay側価格と比較する。

### UC-03
ツールが価格差・概算粗利率・STRを算出する。

### UC-04
ツールが LIST候補 / 保留 / 見送り をスコアベースで表示する。

### UC-05
ユーザーが結果を CSV で保存する。

## 7. 成功条件
- 手作業で相場確認する時間を大幅に削減できる
- 感覚ではなく、数値を見て候補を絞れる
- 毎日のリサーチの起点として使える
