# eBay-Research-Edge 関数仕様書

## プロジェクト概要

**eBay-Research-Edge** は、eBay.com（米国）での販売実績をベースに、日本国内のマーケットプレイス（メルカリ、ハードオフ、Yahoo!フリマ等）から利益の出る商品をリアルタイムでリサーチ・分析し、管理台帳へエクスポートする総合ツールです。

### リサーチフロー

1. **eBay 販売実績の取得 (eBay Browse API)**
   - 米国市場（Site ID: 0）で日本から発送された Sold Listings を取得。
   - 高画質な商品画像を API 経由で取得。

2. **国内マーケットプレイスのライブリサーチ (Playwright)**
   - メルカリ、ハードオフ、Yahoo!フリマをリアルタイムでスクレイピング。
   - 除外フィルタ（備品、オークション形式、中古ジャンク等）による自動ノイズ除去。

3. **インタラクティブ分析レポートの生成**
   - HTMLベースのダッシュボードを自動生成。
   - 国内サイトの商品を選択することで、利益・ROIをリアルタイムにシミュレーション。

4. **Google Sheets へのダイレクトエクスポート (GAS Web App)**
   - 選択した商品をワンクリックで管理用スプレッドシートへ保存。

## 主要モジュール

### 1. generate_research_report.py
**目的**: リサーチの実行、データ集計、HTMLレポートの生成、およびエクスポート受付サーバーの運用。

#### 主要機能
- **API 連携**: eBay REST API (OAuth 2.0) を使用した画像取得。
- **ライブスクレイピング**: 国内3サイトからの最新価格・在庫状況の取得。
- **除外フィルタ**: 
  - `is_relevant()`: キーワードベースでの周辺グッズ（フレーム、空箱、セット品等）の除外。
  - オークション形式の自動検知と除外。
- **Flask サーバー**: レポートからのエクスポートリクエスト（POST）を待ち受け、GASへ転送。

### 2. Google Apps Script (GAS) 連携
**目的**: Pythonから送られたリサーチデータを、認証不要（URLベース）でスプレッドシートへ追記する。

#### 動作
- `doPost(e)`: JSONデータを受け取り、シート「リサーチ結果リスト」へ行を追加。
- カラム構成: 取得日, eBayタイトル, 価格(USD/JPY), 仕入先タイトル, 仕入価格, サイト, URL, 手数料, 送料, 利益, ROI, 画像URL, ステータス。

## 利益計算定義

### eBay 手数料率 (2024年最新)
| Category ID | Category Name | Fee Rate | Notes |
|-------------|---------------|----------|-------|
| 29323 | Trading Cards | 11.35% | ポケモンカード等 (最新) |
| その他 | - | 12.9% | デフォルト |

### 計算式
- **予想利益 (JPY)** = (eBay価格(JPY)) - (国内仕入価格(JPY)) - (eBay手数料(JPY)) - (国際送料(JPY))
- **ROI (%)** = (予想利益 / (国内仕入価格 + 手数料 + 送料)) * 100
